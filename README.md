# Лабораторна робота №7: Хмарне розгортання DevOps-сервісу

## Тема
Хмарне розгортання (deployment) та забезпечення безпеки API-сервісу у середовищі Render.

## Модель
Математичне моделювання біологічного росту бактерій у замкненому середовищі.

## Автор
**Бордіян Микола Павлович**, група АІ-231.

## Опис
У межах роботи виконано деплой (deployment) API-сервісу, розробленого в ЛР5, на хмарну платформу **Render**. Сервіс упаковано у Docker-контейнер, забезпечено публічний HTTPS-доступ та налаштовано безпечну конфігурацію відповідно до вимог DevSecOps.

## Технічні характеристики
| Параметр | Значення |
| :--- | :--- |
| **Хмарна платформа** | Render |
| **Тип розгортання** | Auto-deployment через GitHub |
| **Контейнеризація** | Docker |
| **Endpoint** | `/calculate` |
| **Безпека** | Debug mode: OFF, Port: Environment Variable |

## Як отримати доступ
Публічний URL сервісу: 
`https://devops-231-bordiyan.onrender.com`

**Перевірка роботи (через cURL):**
```bash
curl -X POST [https://devops-231-bordiyan.onrender.com](https://devops-231-bordiyan.onrender.com) \
     -H "Content-Type: application/json" \
     -d '{"N0": 0.1, "S0": 3.0, "t_max": 40}'
```

## Підготовка до DevSecOps
Для забезпечення безпеки сервісу:
- Debug Mode: вимкнено (налаштовано через `app.config['DEBUG'] = False`).
- Environment Variables: порт сервісу зчитується динамічно через `os.environ.get("PORT"`).
- Docker: Dockerfile оптимізовано для безпечної збірки.
- Secrets: паролі та конфіденційні дані відсутні у вихідному коді.

## Результати
Для забезпечення безпеки сервісу:
- Виконано успішний деплой Docker-контейнера у хмарне середовище.
- Забезпечено стабільну роботу API через HTTPS.
- Налаштовано автоматичне оновлення сервісу при кожному push у репозиторій (CI/CD integration).
